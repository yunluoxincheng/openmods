# OpenMods npm 分发迁移计划

## 目标

1. 使用 npm 分发，用户可通过 `npm install -g openmods-ai` 全局安装
2. 用户不需要安装 Bun
3. 将所有 `@opencode-ai/*` 包重命名为 `@openmods-ai/*`

## 参考模式

OpenCode 使用的分发模式：

- 开发时使用 Bun（保留 Bun 特有 API）
- 发布时通过 `Bun.build()` 编译成各平台的单一可执行文件
- npm 包结构：主包 + 平台特定的 optionalDependencies

### 阶段 3：创建 npm 分发结构

**创建文件：`packages/core/script/postinstall.mjs`**

```javascript
#!/usr/bin/env node
import fs from "fs"
import path from "path"
import os from "os"
import { fileURLToPath } from "url"
import { createRequire } from "module"

const __dirname = path.dirname(fileURLToPath(import.meta.url))
const require = createRequire(import.meta.url)

function detectPlatformAndArch() {
  let platform
  switch (os.platform()) {
    case "darwin":
      platform = "darwin"
      break
    case "linux":
      platform = "linux"
      break
    case "win32":
      platform = "windows"
      break
    default:
      platform = os.platform()
  }

  let arch
  switch (os.arch()) {
    case "x64":
      arch = "x64"
      break
    case "arm64":
      arch = "arm64"
      break
    default:
      arch = os.arch()
  }

  return { platform, arch }
}

async function main() {
  try {
    const { platform, arch } = detectPlatformAndArch()
    const packageName = `openmods-ai-${platform}-${arch}`
    const binaryName = platform === "windows" ? "openmods.exe" : "openmods"

    const packageJsonPath = require.resolve(`${packageName}/package.json`)
    const packageDir = path.dirname(packageJsonPath)
    const binaryPath = path.join(packageDir, "bin", binaryName)

    if (!fs.existsSync(binaryPath)) {
      throw new Error(`Binary not found at ${binaryPath}`)
    }

    // 创建链接或复制二进制文件
    const target = path.join(__dirname, "bin", ".openmods")
    if (fs.existsSync(target)) fs.unlinkSync(target)

    if (os.platform() === "win32") {
      fs.copyFileSync(binaryPath, target)
    } else {
      fs.linkSync(binaryPath, target)
      fs.chmodSync(target, 0o755)
    }

    console.log(`openmods binary ready at ${target}`)
  } catch (error) {
    console.error("Failed to setup openmods binary:", error.message)
    process.exit(1)
  }
}

main()
```

**修改文件：`packages/core/bin/openmods`**

```bash
#!/usr/bin/env node

const path = require("path")
const fs = require("fs")
const { spawn } = require("child_process")

const binDir = path.dirname(__filename)
const binaryPath = path.join(binDir, ".openmods")

// 检查二进制文件是否存在
if (!fs.existsSync(binaryPath)) {
  console.error("OpenMods binary not found. Please reinstall the package.")
  process.exit(1)
}

// 运行编译后的二进制文件
const child = spawn(binaryPath, process.argv.slice(2), {
  stdio: "inherit",
  cwd: process.cwd()
})

child.on("close", (code) => {
  process.exit(code || 0)
})
```

## 注意事项

1. **保留 Bun 作为开发工具**：
   - 开发时仍然使用 `bun dev`、`bun test` 等
   - 构建时使用 `Bun.build()`
   - 代码中继续使用 Bun 特有 API

2. **用户端不需要 Bun**：
   - 运行的是编译后的单一可执行文件
   - 二进制文件已包含所有运行时

3. **版本同步**：
   - 主包和所有平台包的版本号必须一致
   - 使用构建脚本自动同步版本

## 执行顺序

1. 批量修改包名和 import 语句
2. 修改构建脚本
3. 创建 postinstall.mjs
4. 修改 bin 入口脚本
5. 更新 package.json 配置
6. 测试构建和安装流程
7. 配置 CI/CD
8. 发布到 npm
